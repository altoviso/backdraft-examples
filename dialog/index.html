Backdraft Dialog Example

This application presents a table of file names, modified timestamps, and sizes, and a rename button that presents a dialog that allows the file name to be changed. If the dialog is OK'd with a non-empty name, then the name is updated. Another little component watches the data behind the file list and outputs changes to that list to demonstrate the underlying data is actually changed...not just what is on the screen.

The program demonstrates declarative composition...notice that, without any knowledge of Backdraft, it is obvious how the rename dialog works.
